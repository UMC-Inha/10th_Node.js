## 1주차 keyword 정리

1. 외래키
- FK(Froeign Key)
- : 테이블과의 관계를 연결 <br /><br />

- Null값 지정 가능 
- PK와 1:1 관계
- FK 보유 테이블 = 하위 테이블
- (PK 보유 테이블 = 상위 테이블)

> 장점
- 데이터 무결성을 보장한다
- 관계 표현이 가능하다
> 단점
- INSERT/UPDATE 시 추가 작업이 필요하기 때문에 성능이 저하된다
- 제약조건으로 인해 유연성이 감소한다 <br /><br />

2. 기본키
- PK(Primary Key)
- : 테이블에서 각 행을 구별하기 위한 값
- 유일성, 최소성을 가짐
- Null값 불가능
- 테이블 당 하나의 PK만 보유 가능

> 장점
- 데이터 식별이 명확하다
- 조회 성능이 향상된다 <br /><br />


3. ER 다이어그램
- : DB 구조를 시각화한 모델

> 구성 요소
- 엔티티(Entity) : 테이블
- 속성(Attribute) : 칼럼
- 관계(Relationship) <br /><br />


4. 복합키
- Composite Key
- : 두 개 이상의 칼럼을 조합한 기본 키

- 단일 칼럼이 아님
- PK의 한 형태
- 조합 시 유일성을 보장함

> 장점
- 데이터 중복 방지에 효과적이다
- 데이터 무결성을 강화한다

> 단점
- 칼럼이 많아져서 설계 복잡도가 증가한다
- 인덱스 크기가 증가한다
- 유지보수가 어렵다


5. 연관관계
- : FK를 통한 테이블 간의 데이터 연결

> 관계 차수
- 1:1 (일대일)
- 1:N (일대다)
- N:M (다대다)
- N:M 관계는 중간에 별도의 테이블(=교차 엔티티)를 만들어 연결해 줘야 함 <br /><br />


6. 정규화
- : 테이블 간 중복 데이터가 없게 하는 설계

> 목표
- 무결성, 일관성, 유연성 향상
- DB 저장 용량 감소

> 정규화 과정
- 제 1 정규화(1NF)
- 각 칼럼이 하나의 값만 갖도록 분해
- 제 2 정규화(2NF)
- 완전 함수 종속(기본키의 부분집합이 결정자가 되지 않음)을 만족하도록 테이블을 분해
- 제 3 정규화(3NF)
- 이행적 종속을 없애도록 테이블을 분해
- 다른 정규화 과정이 더 있지만, 보통 제 3 정규화 과정까지만 거친다 <br /><br />


7. 반 정규화
- : DB 성능 향상을 위해 중복을 허용하는 설계

> 목표
- 조회 속도 향상
- 쿼리 단순화

> 반정규화 기법
- 테이블 수직 분할
- 테이블 수평 분할
- 테이블 병합
